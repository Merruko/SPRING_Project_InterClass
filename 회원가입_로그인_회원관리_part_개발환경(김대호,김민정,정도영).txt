(프로젝트 폴더를 압축한 7z파일과 export한 war파일은 동일한 프로젝트를 담고 있습니다.)


▶프로젝트 환경
IDE      :  이클립스
JAVA    :  1.8
스프링  :  5.0.7 (spring security : 4.1.3)
톰캣     :  8.5
DB       :  ORACLE
mybatis :  3.4.6



▶서버 실행 전 

1. src/main/webapp/WEB-INF/spring/datasource-context.xml 의 사용자명과 비밀번호를 확인해 주십시오.

2. 오라클에서 table과 sequence를 export 하였습니다. import 하거나 프로젝트 내의 [Java Resources/src/main/resources/sql]에 포함된 sql 쿼리문들을 실행해주십시오.

3. 관리자 추가하여야 관리자 권한을 실행할 수 있습니다. import 파일과 sql 쿼리문에 이미 데이터가 포함되어 있습니다만, 만일의 경우를 대비하여 아래의 쿼리문을 기록합니다.
INSERT INTO member (mNum, mId, mPwd, mName, mGender, mBirth, mEmail, mPhone, AUTHORITY, ENABLED)
VALUES ('210101001', 'admin', '12345', '관리자', '남성', '000000', 'admin@test.com', '01012341234', 'ROLE_ADMIN', 1);

4. 톰캣서버 더블클릭 → overview 페이지 하단 modules 클릭 → web module에 프로젝트를 추가해주시고 'path'를 '/'로 변경해주세요.



▶지난번에 피드백 받은 부분에 대한 피드백입니다

0. 마이페이지 화면 레이아웃에 대한 피드백 부분 감사드립니다. 페이지 상단에 nav-bar[내정보, 비밀번호 변경, 정보수정, 회원탈퇴]에서 가장 처음에 사용자가 접근하는 내 정보 페이지를 80% 완료하였습니다. 나머지 부분은 차차 구체적으로 구현하도록 하겠습니다.

1. 회원탈퇴시 아예 삭제처리를 할 건지, 해당 회원이 쓴 글 목록들도 모두 제거할 건지 여부
   -- 이 부분은 현재 회원정보만 완전 삭제 되도록 되어있고, 글 목록들을 제거할지의 여부는 팀원들과 상의예정입니다.
2. 아이디찾기 시 일부 *표시 처리해서 보여주기 
3. 비번 찾기에 연락처나 이메일 확인란 추가
   -- 2번과 3번은 피드백 부분 반영하여 구현 예정입니다.

4. 관리자 회원 관리시 일괄 변경은 막을 것
   -- 아이디부분은 변경 불가하도록 disabled 해놓았습니다. 비밀번호 및 기타 정보의 어느 부분을 disabled 할 것인지 상의 해보겠습니다.
 
5. 해당 seq에 해당하는 아이디에서 옆의 수정 버튼 누를 경우 해당 아이디의 정보수정 페이지로 넘어가서 수정되게끔 처리. 또한 수정자와 수정일자 컬럼도 추가할 것
   -- 현재(21.02.01) 관리자페이지에서 수정버튼 을 누르면 해당 아이디의 정보 수정 페이지로 진입합니다. 수정자와 수정일자 컬럼은 차후 추가하도록 하겠습니다.
